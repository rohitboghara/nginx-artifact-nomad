Architecture Overview

Nomad uses a client-server architecture with a gossip protocol for cluster management

Servers

Manage cluster state, scheduling decisions, and job placement

Clients

Execute tasks and report status back to servers

Jobs

Declarative specifications of workloads to run

Drivers

Execute tasks using Docker, Java, exec, and more
